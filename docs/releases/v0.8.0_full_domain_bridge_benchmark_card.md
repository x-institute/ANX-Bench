# ANX-Bench v0.8.0 Full-Domain Bridge Benchmark Card

## Release Identity

`ANX-Bench v0.8.0` is a frozen candidate full-domain bridge packet for the Wave 8 study `anx_us_2026w08_full_domain_bridge`. Its canonical machine-readable definition is `releases/v0.8.0/manifest.json`. This card is the human-readable companion to that manifest and is bounded by the same frozen item set, checksums, release status, and scoring restrictions.

The purpose of `v0.8.0` is to freeze the planned full-domain bridge administration before outcome inspection. It brings together seven ANX-Bench domains in one preregistered US English online adult bridge packet so that a later evidence release may test whether the candidate item set can support full-domain bridge-readiness claims. Version `v0.8.0` itself does not report observed validation evidence and must not be treated as a scored benchmark release.

## Frozen Candidate Boundary

Version `v0.8.0` is a frozen candidate only. It has no scored items, no domain scores, no cross-domain score, no overall ANX score, and no longitudinal or event-study claims. It also does not promote any new domain to scored status and does not expand the existing citable somatic release line. The four somatic and ambient items appear in Wave 8 only as non-scored bridge anchors for planned comparability and anchor-drift analyses inside the full-domain packet.

The release may be used to reproduce the planned Wave 8 administration, verify that the frozen packet matches its manifest, and prepare the preregistered analyses. It may not be used to calculate an ANX-Bench score for respondents, domains, subgroups, populations, organizations, events, or policies.

## Preregistered Wave 8 Evidence Gates

A later citable `v0.8.1` bridge-readiness evidence release must pass the release-blocking checklist in `docs/releases/v0.8_to_v0.8.1_promotion_gate.md`. The preregistered gates include the following requirements.

| Gate | Release-card summary of required evidence |
| --- | --- |
| Content validity | The completed content-validity dossier must pass before psychometric evidence is reviewed. It must document independent reviewer coverage of domains, constructs, facets, item relevance, item clarity, I-CVI, S-CVI/Ave, unresolved flags, and signoff. |
| Split-sample EFA and CFA | The analytic sample must be split before outcome inspection into independent calibration and confirmation halves. Ordinal EFA must support the preregistered seven-domain structure, and ordinal CFA must confirm a correlated seven-factor model against weaker alternatives without outcome-driven modifications satisfying the gate. |
| Omega | Ordinal omega total must reach the preregistered domain thresholds, with domain-specific reliability evaluated independently. General-factor reliability diagnostics may inform a later aggregate-readiness review but cannot create an aggregate score in Wave 8. |
| IRT | Graded-response or multidimensional IRT models must converge, preserve ordered thresholds, meet discrimination and information requirements, and keep linking error within the preregistered bound. IRT evidence is diagnostic only in `v0.8.0` and cannot introduce theta scoring. |
| DIF | Differential item functioning must be screened for preregistered demographic, exposure, device, sample-source, occupation, and administration-compliance variables where cells permit. Unresolved statistically supported and practically meaningful DIF blocks affected comparison claims and may block bridge readiness. |
| Invariance | Configural, metric, and scalar or threshold invariance must pass preregistered change-fit thresholds for eligible grouping variables. Partial-invariance or IRT-linking fallbacks must be documented, justified, and reviewer approved before any comparability language is allowed. |
| Latent-correlation bounds | Pairwise latent correlations among the seven domains must remain positive, theoretically coherent, and inside the preregistered absolute band of `0.20` to `0.80`. Values outside the band block common-family or simple-aggregation interpretations. |
| Somatic-anchor drift | Somatic bridge-anchor behavior must be compared with the source scoring context. Mean absolute drift must be no greater than `0.20` standard deviations, and no single anchor item may drift more than `0.30` standard deviations unless anchor use is blocked. |
| Reviewer signoff | Independent reviewer signoff must confirm sample integrity, no-event status, item integrity, EFA, CFA, omega, IRT, DIF, invariance, latent-correlation bounds, somatic-anchor drift, aggregate-readiness status, sensitivity analyses, checksum integrity, and claim boundaries. |

## Required Promotion Evidence

Before any full-domain bridge-readiness claim can be made, maintainers must complete and validate all required observed Wave 8 artifacts, including `validation/v0.8/full_domain_bridge/content_validity_dossier.json`, `validation/v0.8/full_domain_bridge/wave8_full_domain_bridge_evidence.json`, and a later `releases/v0.8.1/manifest.json`. The required release-blocking commands are:

```bash
python3 tools/validate_content_validity_dossier.py validation/v0.8/full_domain_bridge/content_validity_dossier.json
python3 tools/validate_full_domain_bridge_evidence.py validation/v0.8/full_domain_bridge/wave8_full_domain_bridge_evidence.json
python3 tools/validate_release.py releases/v0.8.1/manifest.json
```

Passing those commands may support only a citable bridge-readiness evidence statement with the exact claim scope authorized by the later manifest. It remains insufficient by itself for scored-item release, domain scoring, cross-domain scoring, overall ANX scoring, longitudinal comparison, or event-study inference.
